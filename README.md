# Histology Lab QC SQL Analysis



I built this project to practice using SQL on the kind of QC data I work with every day in the histology lab.



It lets me query specimen volumes, IHC slides, error rates, deviations, and turnaround times — the same metrics I check during real-time QC checks every shift.



This is a practical way for me to combine my 8+ years of hands-on CAP/CLIA lab experience with data skills as I move toward QA/QC and clinical data roles.



## Features

- Creates a simple SQLite database with realistic histology QC data

- Runs useful queries I would actually use in a QA/QC or clinical data role

- Shows daily error rates, average turnaround time, technician deviation counts, and total volume summaries

- Demonstrates how SQL can turn raw lab data into actionable quality insights



## Technologies

- SQL (SQLite)

- Python (sqlite3 + Pandas for displaying results)



## How to Run

1. Open `Histology_Lab_QC_SQL_Analysis.ipynb` in Google Colab

2. Run all cells

3. The notebook creates the database and prints 4 useful QA/QC reports
